Weighting: These define how much a NSI/SSI/GSI is worth (in that order). There are default weights which describe "to check" and "to counter". (These might not show on Safari). See the About page for more information if you want to change these from the defaults (Warning: it's complicated).

Optimize: Switches beween NSI/SSI/GSI data and NSI to/SSI to/GSI to data. "By" data considers how good a team is at checking/countering, and "To" data considers how good the metagame can check/counter the team. Note that both the weighting and the view affect the interpretation of the output:

  • [Check Weighting, By]: Finds teams which check the most: Generates the best teams offensively
  • [Counter Weighting, By]: Finds the teams which counter the most: Genereates the best teams defensively
  • [Check Weighting, To]: Finds the teams which are checked the least: Generates the teams that are the hardest to sweep
  • [Counter Weighting, To]: Finds the teams which are countered the least: Generates the best teams for breaking stall

Blacklist/Whitelist: Enter Pokemon Sets into these fields. Each entry must exactly match with a set in the Checks Compendium build data so use the autocomplete suggestions to help. (The suggestions may not work on Safari). Click "Add Blacklist/Whitelist" to input more Pokemon. Refresh or return to this page to reset. Pokemon inputted here are blacklisted from the generation algorithm (meaning that they won't be suggested as teammates).

Blacklist/Whitelist Mode: If whitelist mode is enabled, only inputted Pokemon are used by the team generator.

Blacklist Rain: Puts rain Pokemon on the generation blacklist if in blacklist mode.

Make Usage Blind: Normally, the mkmt weights Pokemon by usage by multiplying a Pokemon's set usage with its check weighting. The thinking is that it is better to counter 1 Pokemon with 5% usage than 2 Pokemon with 2% usage because the team is more likely to have a favorable matchup against a random team on the ladder (5 > 2*2). Sometimes, this is unwanted or does not make sense (such as when considering a small fraction of the metagame or counterteaming, or when building a team where ladder usage is not representative of what the team is expected to face).

Filter Duplicate Species and Megas: This removes teams containing more than one Mega Pokemon or more than one Pokemon of the same species.
